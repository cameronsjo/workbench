# Pre-Presentation Checklist

Use this comprehensive checklist before every executive presentation to ensure quality and effectiveness.

## Content Quality (Narrative Structure)

### What/Why/Next Framework

- [ ] **WHAT**: Current state clearly stated with specific metrics
- [ ] **WHAT**: Metrics aligned with CEO priority (Growth/Technology/Workforce/Financial)
- [ ] **WHAT**: Current trajectory clear (on track/ahead/behind targets)
- [ ] **WHY**: Root cause analysis is data-driven, not speculative
- [ ] **WHY**: Multiple supporting data points provided (not just one)
- [ ] **WHY**: Failures are depersonalized (focus on problem, not blame)
- [ ] **NEXT**: Clear, specific recommendation (not vague suggestion)
- [ ] **NEXT**: Expected outcomes are quantified with timeline
- [ ] **NEXT**: Investment required is specified (budget, headcount, resources)
- [ ] **NEXT**: Success metrics defined for tracking progress

### Decision Clarity

- [ ] Decision required is explicitly stated (not implied)
- [ ] Decision type is clear (approval, prioritization, resource allocation, direction, awareness)
- [ ] Timeline for decision is specified with reasoning
- [ ] Options are presented if multiple paths exist
- [ ] Trade-offs between options are clearly explained
- [ ] Next steps are defined based on each possible decision

### Strategic Alignment

- [ ] Every key metric connects to a CEO priority
- [ ] Narrative explains why this matters to company, not just department
- [ ] Language mirrors CEO and ELT terminology (not department jargon)
- [ ] Stakeholders affected by this decision are identified
- [ ] Cross-functional implications are addressed

---

## Design Quality (Visual Presentation)

### Slide Structure

- [ ] Title slide includes initiative name, date, presenter
- [ ] Executive summary slide states key insight and decision required
- [ ] Main deck is 7-12 slides (for 30-minute meeting)
- [ ] Each slide follows "one slide, one idea" principle
- [ ] Slide titles state insights, not just topics
  - ✅ "Premium Leads Increase Pipeline 35%"
  - ❌ "Lead Generation Results"
- [ ] Logical flow: Current state → Analysis → Recommendations → Implementation

### Bullet Points and Text

- [ ] Each slide has 3-5 bullets maximum (never more than 5)
- [ ] Bullets are concise (5-10 words each, not full sentences)
- [ ] No dense paragraphs or blocks of text
- [ ] Font size is 18pt or larger for body text (24pt+ for titles)
- [ ] Text formatting is minimal (bold for emphasis only, no excessive highlighting)

### Visual Elements

- [ ] Charts and graphs support the narrative (not just decoration)
- [ ] Charts are simple and easy to understand at a glance
- [ ] Chart types are appropriate for data (line for trends, bar for comparisons, etc.)
- [ ] All charts have clear axis labels with units
- [ ] Data labels are included on charts when values matter
- [ ] Colors are used strategically to convey meaning (not random)
- [ ] Color scheme is consistent across all slides
- [ ] Visuals are high contrast and colorblind-accessible
- [ ] No 3D effects, excessive decoration, or clip art

### Typography and Formatting

- [ ] Sans-serif fonts used throughout (Arial, Calibri, Helvetica)
- [ ] No more than 2 font families in entire deck
- [ ] Consistent formatting (headers, bullets, spacing) across all slides
- [ ] Sufficient white space (not cramped or cluttered)
- [ ] Text is left-aligned for readability (not centered)
- [ ] Numbered or bulleted lists are consistent in style

---

## Language Quality (Clarity and Accessibility)

### Jargon and Acronyms

- [ ] Jargon audit completed with someone outside your department
- [ ] All acronyms are defined on first use (or eliminated)
- [ ] Department-specific terminology is translated to plain language
- [ ] Technical concepts have analogies for non-technical executives
- [ ] No unexplained abbreviations or shorthand

### Tone and Voice

- [ ] Tone is confident but not arrogant
- [ ] Failures are presented analytically, not defensively
- [ ] Language is direct and clear, not vague or hedging
- [ ] Emotional tone (surprised/inspired/reassured/concerned) is appropriate
- [ ] Writing is active voice and present tense
- [ ] Avoids phrases like "we struggled" or "we couldn't" (depersonalize)

### Readability

- [ ] Slides pass the "glance test" (understandable in 5 seconds)
- [ ] Sentence case used (not ALL CAPS)
- [ ] Numbers are formatted consistently (e.g., $2.3M not $2,300,000)
- [ ] Percentages and ratios are clear (23% increase, 4:1 ratio)
- [ ] Dates and timeframes are explicit (Q3 2024, not "next quarter")

---

## Data Quality (Accuracy and Consistency)

### Data Accuracy

- [ ] All metrics are accurate and verified
- [ ] Data sources are cited in footnotes
- [ ] Calculation methods are documented (in appendix if complex)
- [ ] Timeframes are consistent across all slides
- [ ] Comparisons use same baseline (apples-to-apples)
- [ ] Projections are based on historical data or industry benchmarks

### Data Presentation

- [ ] Units are specified (dollars, users, days, percentage, etc.)
- [ ] Scale is consistent (don't mix millions and thousands on same chart)
- [ ] Y-axis starts at appropriate point (not exaggerating trends)
- [ ] Trends show sufficient time period (not cherry-picked dates)
- [ ] Key data points are annotated on charts
- [ ] Outliers or anomalies are explained

### Supporting Evidence

- [ ] Claims are backed by data, not opinions
- [ ] Multiple data points support key conclusions
- [ ] Competitive benchmarks included where relevant
- [ ] Historical context provided for trends
- [ ] Statistical significance noted if applicable

---

## Appendix Quality (Supporting Materials)

### Appendix Structure

- [ ] Appendix exists and is well-organized
- [ ] Appendix slides are numbered (e.g., A1, A2 or 11, 12)
- [ ] Appendix is referenced from main deck where appropriate
- [ ] Sections are clearly labeled (Methodology, Financial Model, FAQ, etc.)
- [ ] Navigation is easy (can quickly find specific supporting slide)

### Appendix Content

- [ ] Detailed methodology documented
- [ ] Data sources and calculations explained
- [ ] Alternative scenarios or analyses included
- [ ] Competitive benchmarking data provided
- [ ] Implementation project plan with timeline
- [ ] Risk assessment and mitigation strategies
- [ ] FAQ with anticipated questions and answers
- [ ] Supporting charts and tables (too detailed for main deck)

### Appendix Preparation

- [ ] You know which appendix slide answers which likely question
- [ ] Appendix slides are just as polished as main deck
- [ ] Appendix slides can stand alone if referenced
- [ ] You can navigate to any appendix slide within 5 seconds

---

## Stakeholder Management (Pre-Wiring)

### Pre-Meeting Preparation

- [ ] Key stakeholders identified (who must approve, who must support)
- [ ] 1:1 meetings scheduled with decision makers before presentation
- [ ] Executive summary shared with stakeholders in advance
- [ ] Feedback from pre-meetings incorporated into presentation
- [ ] Potential objections identified by stakeholder
- [ ] Responses to objections prepared
- [ ] Champion identified who will support your recommendation

### Stakeholder Analysis

- [ ] You understand each executive's priorities and concerns
- [ ] You know how this decision affects each stakeholder's area
- [ ] You've addressed concerns specific to each key stakeholder
- [ ] You have allies who will speak up in support
- [ ] You know who might object and why

---

## Practical Logistics

### Timing

- [ ] Presentation fits within allocated time (leave buffer for questions)
- [ ] You've practiced with a timer
- [ ] You know which slides can be skipped if running short on time
- [ ] You've prepared a 60-second elevator version if meeting is cut short

### Technical Readiness

- [ ] Deck is saved in multiple formats (PowerPoint, PDF, Google Slides)
- [ ] File is named clearly with date (e.g., "Executive_AI_Strategy_2024-08-15.pptx")
- [ ] Deck displays correctly on different screen sizes
- [ ] Animations work properly (if used)
- [ ] Links to appendix slides function correctly
- [ ] Backup copy exists (email to yourself, USB drive, cloud storage)

### Delivery Preparation

- [ ] You've practiced presenting aloud at least twice
- [ ] You can explain each slide without reading from it
- [ ] You know your key points if slides are lost (technical failure)
- [ ] You have printed notes or talking points (if needed)
- [ ] You've anticipated tough questions and prepared responses

---

## Quality Assurance Tests

### The Elevator Test

- [ ] You can summarize the entire presentation in 60 seconds
- [ ] Elevator pitch includes What, Why, Next, and decision required
- [ ] Someone unfamiliar with the topic understands the elevator pitch

### The Outside Perspective Test

- [ ] Someone outside your department reviewed the deck
- [ ] They understood every term and concept
- [ ] They could explain the key insight to someone else
- [ ] They identified confusing or unclear elements (which you've fixed)

### The Decision Clarity Test

- [ ] Someone unfamiliar with the presentation can answer:
  - What decision is being requested?
  - What are the options?
  - What happens next based on each decision?
  - When is the decision needed?

### The Glance Test

- [ ] Each slide is understandable in 5 seconds or less
- [ ] Key insight is obvious without presenter speaking
- [ ] Charts and visuals are immediately clear

### The Strategic Relevance Test

- [ ] For each key metric, you can answer:
  - Which CEO priority does this connect to?
  - How does this affect other executives besides me?
  - What's the company-level impact?
- [ ] An executive from a different department would understand the significance

---

## Final Review

### Content Completeness

- [ ] All required sections are present (current state, analysis, recommendations, next steps)
- [ ] No placeholders or "TBD" items remain
- [ ] All references and citations are complete
- [ ] Contact information included if follow-up needed

### Polish and Professionalism

- [ ] No typos or grammatical errors
- [ ] Consistent capitalization and punctuation
- [ ] Company branding and style guide followed (if applicable)
- [ ] Professional color scheme and design
- [ ] Page numbers included
- [ ] Date on title slide is correct

### Legal and Compliance

- [ ] No confidential information exposed to wrong audience
- [ ] Competitive intelligence is from public sources
- [ ] Financial data complies with disclosure policies
- [ ] No misleading or exaggerated claims

---

## Post-Presentation Planning

### Preparation for Questions

- [ ] List of 10 tough questions you might be asked
- [ ] Prepared response for each question
- [ ] Know which appendix slide supports each answer
- [ ] Ready to say "I don't know, I'll follow up" if asked something unexpected

### Follow-Up Plan

- [ ] Process defined for getting decision if not made in meeting
- [ ] Follow-up meeting scheduled if needed
- [ ] Owner assigned for next steps
- [ ] Timeline for follow-up communication established

## Emergency Pre-Flight Check (5 Minutes Before)

If you only have 5 minutes, check these critical items:

- [ ] File opens correctly and displays properly
- [ ] Slide count matches time available
- [ ] Executive summary slide clearly states decision required
- [ ] All data is accurate and cited
- [ ] No typos on key slides (title, executive summary, recommendations)
- [ ] Appendix is ready and you can navigate to it
- [ ] You know your 60-second elevator pitch
- [ ] You have backup of presentation file

---

## Continuous Improvement

After the presentation:

- Document what worked well
- Note questions you weren't prepared for
- Add those questions to your FAQ appendix
- Refine your narrative based on feedback
- Save effective slides for reuse in future presentations
- Ask a trusted executive: "How could I have made this more compelling?"

**Remember:** Executive data storytelling is a skill that improves with practice and feedback. Use this checklist every time to build excellence.
